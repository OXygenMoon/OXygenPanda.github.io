新关键词 using (Alias Template & Type Alias)

新关键词 using

namespace

  • using-指示 : using namespace std;

    引入整个命名空间的代码

    using namespace std 会引入命名污染，尽量放在较小的作用域中偷懒，在较大作用域中应该使用 using-声明

  • using-声明 : using std::cout, std::endl;

    引入某命名空间中的一些对象、函数

class members

  • using-声明

    ​ 在某个类中使用其他类的对象或方法

    class base {
public:
    base(int num,string name):b_num(num),b_name(name){}
    void print(){std::cout << b_num << b_name << std::endl;}
public:
	int b_num;
    string b_name;
}; //base end

class drived : public base {
public:
    drived(int num1,int num2,string name1,string name2):d_num(num1),d_name(name1),base(num2,name2){}
    
	using base::print; //使用 print 替换 base::print 的写法
    
    void d_print(){
    	print(); //此处调用
    	std::cout << d_num<<d_name << std::endl;}
private:
	int d_num;
	string d_name;
}; //drived end

Alias Template

化名 (using 语法)

template<typename T>
using MapString = map<T,char*>;
MapString<int> numberString;   //解析成 map<int,char*> numberString;

个人感觉，这用法有一种提前绑定多模板参数中的若干参数，有一种偏特化的意味。侯捷老师提供了一个案例，必须得使用 alias template .

//侯捷老师的例子
template<typename T, template<class> class Container>
class XCLs{
private:
		Container<T> c;
public:
		XCLs(){}
};

//!
XCLs<int,vector> xcls;   //报错,因为vector容器实际上需要两个参数,这样传入只能确认一个参数

template<T>
using vec = vector<T,allocator<T>>;
XCLs<int,vec> xcls; //正确,提前绑定第二参数

因为 vector 模板类本身有两个模板参数 : _Tp 和 alloc，而 alloc 是有默认参数的。所以，平时使用时，我们往往缺省第二参数只指定第一个模板参数类型。而 5:Container<T> c 只指定了第一参数，因此在 2: 传入 Container 时就提前确定第二参数，因此有了 14: Alias Template 的操作。

Type Alias

类似于 typedef 的语法。(结果来看和typedef一样)

typedef void(*func1)(int,int); //func1 绑定为 void x(int,int) 型函数指针
using func2 = void(*)(int,int); //结果同 1:

//演示1
void example(int a, int b);
func1 fn1 = example;
func2 fn2 = example;

//演示2
template<typename T>
class vector{
public:
    using ValueType = T;
    using reference = T&;
    using iterator  = T*;
    using pointer   = T*;  //vector中的实现还是typedef
}
